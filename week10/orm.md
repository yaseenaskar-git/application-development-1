# ORM Concept Reflection

## 1. What problem does raw SQL create in large applications?

Raw SQL can make the code very complex in terms of clarity and maintainability, increasing chances of error vulnerabilities.

## 2. What is an ORM in your own words?

ORM stands for "object-relational mapping", and it is the practice of interacting with data in databases via programming languages. More specifically, using object-oriented programming languages instead of writing in SQL.

## 3. What does an ORM replace or simplify?

ORM can simplify and replace SQL when dealing with common functions of the database, such as creating, reading, updating, etc.

## 4. When would you NOT want to use an ORM?

ORM would not be used in non-relational databases. In addition, ORM would not be used when complex or high performance functions are required from the database, and an example of this is complex queries, which could be manipulated with more control with SQL.
